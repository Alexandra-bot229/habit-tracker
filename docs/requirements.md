# Документ требований к проекту «Трекер привычек»

## 1. Проблема (какую боль решает приложение)

Многие люди хотят сформировать полезные привычки: делать зарядку по утрам, пить воду, читать книги, учить иностранные слова. Но без системы отслеживания сложно понять, насколько регулярно выполняются эти действия. Человек забывает, делал ли он что-то сегодня, теряет мотивацию и бросает начатое.

**Решаемая проблема:** Отсутствие простого инструмента для ежедневного отслеживания привычек и визуализации прогресса.

## 2. Целевая аудитория

| Сегмент | Описание |
|---------|----------|
| Студенты | Нуждаются в организации расписания и самодисциплины |
| Люди, формирующие привычки | Кто хочет внедрить полезные привычки в свою жизнь |
| Любители саморазвития | Кто ведёт дневник привычек и хочет видеть прогресс |

**Портрет пользователя:**
- Возраст: 16–35 лет
- Есть компьютер или смартфон с доступом в интернет
- Хочет отслеживать 1–5 привычек ежедневно
- Ценит простоту и наглядность

## 3. Функциональные требования (минимум 5)

| № | Требование | Приоритет |
|---|------------|-----------|
| 1 | Пользователь может зарегистрироваться и войти в систему (у каждого свои данные) | Высокий |
| 2 | Пользователь может добавить привычку (название, описание, целевое количество раз в день) | Высокий |
| 3 | Пользователь может отметить привычку как выполненную за сегодня | Высокий |
| 4 | Пользователь видит прогресс выполнения привычки в процентах | Средний |
| 5 | Пользователь может посмотреть историю выполнения привычки за последние 7 дней | Средний |
| 6 | Пользователь может искать привычки по названию | Низкий |
| 7 | Пользователь может удалить привычку (с подтверждением) | Средний |
| 8 | Пользователь может выйти из системы | Низкий |

## 4. Нефункциональные требования

| № | Требование | Описание |
|---|------------|----------|
| 1 | Простота использования | Интерфейс интуитивно понятен, не требует обучения |
| 2 | Скорость | Страницы загружаются менее чем за 1 секунду |
| 3 | Доступность | Работает в любом современном браузере (Chrome, Firefox, Edge) |
| 4 | Лёгкость установки | Достаточно склонировать репозиторий и установить зависимости |


## 7. Технологический стек

| Компонент | Технология | Причина выбора |
|-----------|------------|----------------|
| Язык | Python 3.12 | Простота, большой опыт, встроенная поддержка SQLite |
| Веб-фреймворк | Flask | Лёгкий, быстрый, не требует много кода |
| База данных | SQLite | Не нужен отдельный сервер, один файл, встроен в Python |
| Интерфейс | Bootstrap 5 | Быстрая разработка, адаптивность, красиво "из коробки" |
| Тестирование | pytest | Простой синтаксис, удобные фикстуры |
| Контроль версий | Git + GitHub | Хранение кода, отслеживание изменений |
